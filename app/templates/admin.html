    Setting up Microsoft Entra ID (Azure AD) SSO
    

                
    
                    
    
                        
    
                            
                        
    
                        
    
                            
    
                                
      
                                    
    1. Sign in to the Azure Portal
      2. 
                                    
    2. Navigate to Microsoft Entra ID (Azure Active Directory)
      3. 
                                    
    3. Go to Enterprise Applications  + New application
      4. 
                                    
    4. Click + Create your own application
      5. 
                                    
    5. Name it "Lab Document Intelligence" or similar
      6. 
                                    
    6. Select "Integrate any other application you don't find in the gallery (Non-gallery)"
      7. 
                                    
    7. Click Create
      8. 
                                
    
                            
    
                        
    
                    
    

                    
    
                        
    
                            
                        
    
                        
    
                            
    
                                
      
                                    
    1. In your Enterprise Application, go to Single sign-on
      2. 
                                    
    2. Select SAML
      3. 
                                    
    3. In Basic SAML Configuration, enter:
                                        
        
                                            
      • Identifier (Entity ID): Copy from the SSO Configuration panel
        • 
                                            
      • Reply URL (ACS URL): Copy from the SSO Configuration panel
        • 
                                        
      
                                    
      4. 
                                    
    4. Download the Federation Metadata XML URL
      5. 
                                    
    5. Configure claims as needed (email, name, groups)
      6. 
                                
    
                            
    
                        
    
                    
    

                    
    
                        
    
                            
                        
    
                        
    
                            
    
                                
      
                                    
    1. In your Enterprise Application, go to Provisioning
      2. 
                                    
    2. Click Get started
      3. 
                                    
    3. Set Provisioning Mode to Automatic
      4. 
                                    
    4. Under Admin Credentials:
                                        
        
                                            
      • Tenant URL: Copy from the SCIM Configuration panel
        • 
                                            
      • Secret Token: Copy from the SCIM Configuration panel
        • 
                                        
      
                                    
      5. 
                                    
    5. Click Test Connection to verify
      6.
